# PaperGrid Landing Page — Full Premium Overhaul (Approach C)

## Overview
Rebuild every section of the PaperGrid AI Notebook landing page to editorial-magazine quality with cinematic continuity from the 3D hero through all content sections. The current state: the hero + StatsStrip transition is polished (dark-to-parchment gradient, scrub-driven counting numbers, paper dust particles), but all sections below StatsStrip feel flat and disconnected from the cinematic 3D experience above.

## Goals
1. Extend the storytelling narrative from hero through ALL sections (no "flat after hero" problem)
2. Make every section interactive, not just visually attractive
3. Eliminate dead weight sections and restructure for better conversion flow
4. Create a unified color flow and design system across all sections
5. Add signature interaction patterns (ink ripple, magnetic buttons, cursor tracking)
6. Achieve editorial-magazine quality comparable to Linear.app, Raycast.com, Framer.com landing pages

## Tech Stack Context
- React 19 + TypeScript + Vite
- GSAP + ScrollTrigger for scroll animations
- React Three Fiber (R3F) for 3D elements
- Three.js for materials, geometry
- Tailwind CSS v4 with custom @theme tokens
- Existing 3D components: HeroNotebook.tsx, HeroScene.tsx, BookCoverScene.tsx, FloatingPapers.tsx (disabled), PostEffects.tsx, PaperMaterial.tsx

## Current Landing Page Structure (in order)
1. NavBar - sticky navigation
2. HeroSection - pinned scroll-driven 3D notebook with 3 text phases (350vh)
3. StatsStrip - dark-to-parchment gradient transition with counting numbers (DONE, polished)
4. PaperStylesSection - 10 paper type grid with fan entrance
5. AIFeatureSection - AI layout generation showcase
6. BlockTypesBento - bento grid of 12 block types
7. AestheticsSection - ASMR sounds + stickers/washi tape features
8. HowItWorksSection - 3-step getting started guide
9. TestimonialsSection - user testimonials
10. FinalCTA - final call to action
11. LandingFooter - footer

## TASK 2: Cursor-Tracked 3D Notebook Tilt

### Problem
HeroNotebook.tsx accepts a `hovered` prop but never uses it. The 3D notebook sits passively — no mouse interactivity.

### Requirements
- Track mouse/cursor position relative to the hero viewport
- Pass normalized cursor position (-1 to 1, x and y) to HeroNotebook via a shared ref (no re-renders)
- In HeroNotebook's useFrame: apply subtle tilt based on cursor position
  - rotateX: cursor.y * 0.08 (max 0.08 radians, ~4.5 degrees)
  - rotateY: cursor.x * 0.12 (max 0.12 radians, ~7 degrees)
- Smooth lerp on cursor tracking (same Math.pow(0.001, delta) pattern)
- Tilt fades out as book opens (multiply by `1 - open` factor)
- Mobile: disable cursor tracking (touch doesn't have persistent cursor)
- On hover: slight scale increase (1.0 -> 1.03) with smooth transition

### Files
- `packages/web/src/components/landing/HeroSection.tsx` (mouse tracking, pass ref)
- `packages/web/src/components/three/landing/HeroNotebook.tsx` (consume cursor ref, apply tilt)
- `packages/web/src/components/three/landing/HeroScene.tsx` (pass through cursor ref)

## TASK 17: Performance Audit and Optimization

### Problem
Adding 3D elements, animations, and interactions increases performance risk. Need to ensure 60fps on mid-range devices.

### Requirements
- Add FPS monitor (development only) — shows current FPS in corner
- Implement auto-degradation:
  - If FPS drops below 45 for 2+ seconds: disable FloatingPapers
  - If FPS drops below 30 for 2+ seconds: disable PostEffects, simplify scroll animations
- Lazy load heavy components (AmbientCanvas, ClosingNotebook) with React.lazy + Suspense
- Ensure all GSAP ScrollTrigger instances are cleaned up on unmount
- Add `will-change` hints to animated elements
- Verify all `useFrame` loops check `delta` bounds (clamp to prevent spiral of death)

### Files
- `packages/web/src/hooks/usePerformanceMonitor.ts` (NEW)
- `packages/web/src/components/LandingPage.tsx` (lazy loading, perf hooks)
- All components with GSAP (verify cleanup)

## TASK 18: Reduced Motion and Accessibility

### Problem
All new animations need reduced-motion alternatives. Interactive elements need keyboard accessibility.

### Requirements
- Every new animation has a `prefers-reduced-motion: reduce` override in globals.css
- Typewriter effect: shows full text immediately
- Ink progress line: hidden
- FloatingPapers: hidden
- Carousel: no auto-scroll, shows as static grid
- All interactive elements (magnetic buttons, bento cards) have focus-visible styles
- Carousel is keyboard-navigable (arrow keys)
- ARIA labels on decorative elements (aria-hidden="true" on ink line, floating papers)
- Screen reader: skip decorative 3D canvases

### Files
- `packages/web/src/styles/globals.css` (reduced motion overrides)
