# Specification: AI Layout Generation v2 — Richer Blocks + Smarter AI

**Track ID:** ai-layout-v2_20260330
**Type:** Feature
**Created:** 2026-03-30
**Status:** Draft

## Summary

Upgrade the AI layout generation system with richer block data models and smarter AI pre-filling so generated layouts are functional and content-rich instead of empty shells. Calendar gets events per day, WeeklyView gets tasks per day, and the AI generates contextual content based on the user's prompt.

## Context

PaperGrid generates notebook layouts from natural language prompts via Gemini API. The current system has 17 block types with Zod-validated schemas, few-shot reference matching, and structured JSON output. However, generated layouts suffer from two problems: (1) blocks like Calendar are display-only with no actionable data, and (2) the AI fills blocks with empty placeholders instead of contextual content derived from the prompt.

## User Story

As a PaperGrid user, I want AI-generated layouts to contain rich, pre-filled content relevant to my prompt so that the generated notebook is immediately useful instead of requiring me to fill in everything from scratch.

## Acceptance Criteria

- [ ] Calendar block supports per-day events (title + optional color), not just highlighted day numbers
- [ ] Calendar AI generation pre-fills events based on prompt context (e.g., "finals week" → exam dates)
- [ ] WeeklyView block supports per-day task lists, not just a single text area
- [ ] WeeklyView AI generation pre-fills day tasks from prompt context
- [ ] AI generates contextual content for ALL block types (TimeBlock entries, HabitTracker habits, GoalSection goals) based on prompt
- [ ] Multi-page generation: a single prompt can produce multiple coordinated pages (e.g., "weekly planner" = overview + 7 daily pages)
- [ ] All schema changes validated with Zod in @papergrid/core
- [ ] All existing reference layouts updated to use new data models
- [ ] AI prompt rules updated to instruct Gemini on new capabilities
- [ ] No regressions in existing layout generation

## Technical Notes

- Vertical slice approach: upgrade schema → renderer → AI prompt → reference layouts per block type
- Calendar events schema: `events: Array<{day: number, title: string, color?: string}>`
- WeeklyView tasks schema: `tasks: Array<{text: string, checked: boolean}>` per day
- Multi-page: new endpoint or extended response schema returning `pages[]` instead of single page
- Gemini responseSchema must be updated alongside prompt rules
- Reference layouts in http.ts must be updated to use new data shapes
